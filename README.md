# OI Scanner Bot — исправленная версия

Депозит **500$**, плечо **10x**.

---

## Что было исправлено (критично)

### 1. Максимальное плечо → фиксированное 10x
**Было:** `set_leverage(symbol, info["max_leverage"])` — ставилось максимальное плечо биржи (до 75-100x).

**Почему опасно:** при плече 50x ликвидация наступает на движении ~2%. Стоп-лосс, стоявший на 30%, не срабатывал **никогда** — позицию просто ликвидировало.

**Стало:** плечо берётся из `LEVERAGE` (по умолчанию 10) и ограничивается лимитом инструмента. Ликвидация ~10%, стоп 2% — срабатывает в 5 раз раньше.

### 2. Стоп-лосс 30% → 2%
**Было:** `AUTO_HARD_SL_PCT = 30.0`, `AUTO_PULLBACK_SL_PCT = 30.0`
**Стало:** `2.0` и `1.2`

Соотношение риск/прибыль:

| | было | стало |
|---|---|---|
| STANDARD | TP 2.15% / SL 30% = **1:0.07** | TP 3% / SL 2% = **1:1.5** |
| Безубыточный win rate | **93.3%** (недостижимо) | **40%** (реально) |
| PULLBACK | TP 1.15% / SL 30% | TP 1.8% / SL 1.2% |

### 3. TP/SL считались от цены ДО входа
**Было:** цели рассчитывались от `last_price`, но ордер маркетный — реальная цена исполнения другая. При проскальзывании 1% фактический TP становился +1.14% вместо +2.15%.

**Стало:** после подтверждения позиции читается фактическая `avgPrice`, и TP/SL **переставляются** от неё через `/v5/position/trading-stop`.

### 4. Не проверялось, что стоп реально стоит
**Стало:** после входа вызывается `verify_position_protected()`. Если стопа на бирже нет — позиция **закрывается сразу**, незащищённую не держим.

### 5. Состояние терялось при редеплое
**Было:** JSON-файлы рядом с кодом (`auto_state.json`), в `.gitignore`, на Railway файловая система эфемерная. При редеплое `active_positions` обнулялся, а позиции на бирже **оставались открытыми** и осиротевали.

**Стало:** все файлы пишутся в `DATA_DIR` (по умолчанию `/data` — смонтированный volume).

### 6. Один стоп = дневной лимит
**Было:** позиция 1000$, стоп 30% → убыток 300$ при лимите 300$. Блокировка после **первого** стопа.
**Стало:** позиция 250$, стоп 2% → убыток 5$ при лимите 25$. Запас на 5 стопов.

## Правки в сигналах (из твоего документа)

**`try_surge`** — защита от short squeeze:
- OI должен расти и за 4h (`oi_change_4h >= 4.0`), а не только за 1h
- цена уверенно выше EMA50 (запас 0.5%), а не упирается в неё
- звёзды: 2★ требует OI 1h ≥ порог×1.8 **и** OI 4h ≥ 8%

**`try_pullback`** — защита от отскоков в конце тренда:
- EMA50 должна **расти** (сравнение с EMA50 10 баров назад)
- отскок должен иметь силу: минимум +0.6% от локального минимума
- звёзды: 2★ требует OI 24h ≥ порог×1.6 **и** OI 1h > 2%

## Установка на Railway

1. Загрузи все файлы в репозиторий.
2. Создай сервис из репозитория.
3. **Смонтируй volume на `/data`** — без этого состояние потеряется при редеплое.
4. Задай переменные:

| Переменная | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота |
| `TELEGRAM_CHAT_ID` | твой chat id |
| `BYBIT_API_KEY` | ключ |
| `BYBIT_API_SECRET` | секрет |
| `DATA_DIR` | `/data` |

Остальные параметры имеют разумные дефолты, менять не обязательно.

### Настраиваемые параметры риска

| Переменная | По умолчанию | Смысл |
|---|---|---|
| `DEPOSIT_USD` | 500 | депозит |
| `LEVERAGE` | 10 | плечо (не выше лимита инструмента) |
| `POSITION_SIZE_USD` | 250 | номинал позиции |
| `AUTO_TP_PCT` | 3.0 | тейк STANDARD/SURGE |
| `AUTO_HARD_SL_PCT` | 2.0 | стоп STANDARD/SURGE |
| `AUTO_PULLBACK_TP_PCT` | 1.8 | тейк PULLBACK |
| `AUTO_PULLBACK_SL_PCT` | 1.2 | стоп PULLBACK |
| `DAILY_LOSS_LIMIT_USD` | 25 | дневной лимит убытка |
| `MAX_AUTO_POSITIONS` | 2 | позиций одновременно |

## Важно перед запуском

Авто-торговля **выключена по умолчанию** (`auto_enabled: False` в состоянии). Включается командой в боте.

Рекомендую сначала:
1. Запустить и посмотреть, какие сигналы приходят — без авто-входа.
2. Убедиться, что volume смонтирован и состояние переживает редеплой.
3. Проверить на demo-ключах Bybit, что ордера ставятся и стопы видны в интерфейсе биржи.
4. Только потом включать авто-торговлю на реальных.

Стратегия (STANDARD / SURGE / PULLBACK на OI) **не проходила бэктест** — исправлены только риск и исполнение. Есть ли у сигналов преимущество, покажет статистика.
